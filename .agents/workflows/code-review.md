# Code Review Workflow

## Step 1: Understand the Change

- Review the purpose of the change
- Check related issues or tasks

## Step 2: Review Architecture

Verify:

- Proper file placement
- Reusable components
- Clean separation of concerns

## Step 3: Review Code Quality

Check:

- Naming conventions
- Function complexity
- Code duplication
- Readability

## Step 4: Security Check

Look for:

- SQL injection
- Unvalidated inputs
- Hardcoded secrets
- Unsafe authentication logic

## Step 5: Performance Check

Review:

- Database queries
- Loops and recursion
- Network calls

## Step 6: Suggest Improvements

Provide:

- Specific recommendations
- Simpler alternatives
- Best practices
